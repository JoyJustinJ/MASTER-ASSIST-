8. JAVASCRIPT BEHAVIOR
========================

A. Form Submit (login-api.js):
   - Prevents default
   - Changes button icon to spinning cog (fa-cog fa-spin)
   - Fires parallel POST to newapp/auth/login (Laravel API) — async/await
   - After 1500ms delay: AJAX POST to login1.php
   - Handles responses:
     status "200" → SweetAlert success → redirect to welcome.php
                  → clears localStorage: get_pay_mode_1, country_state, state_district
     status "203" → OTP required → GET api-loginotp.php?Token=X
                  → On success: set #tokenid, set #replace_mobno, fadeIn popup, start timer(120)
     status "209" → Multi-device conflict → SweetAlert confirm "Logout from all device"
                  → If confirmed: GET logout_from_all_device.php?Token=X&status=Ajax
     other        → SweetAlert error with response.message

B. OTP Timer:
   - Starts at 120 seconds (2 minutes)
   - Counts down MM:SS format
   - When reaches 0: shows "Resend OTP" link
   - timerOn flag stops timer when popup is closed

C. Password Toggle:
   - Click .js-password-toggle → toggles type="password"/"text"
   - Shows/hides fa-eye and fa-eye-slash icons

D. Forgot Password Modal:
   - #forgotPasswordToggle click → show modal
   - Modal close button → hide modal + reset
   - Step 1: Enter mobile → POST to forgot_password.php action="send_otp"
   - Step 2: (shown after OTP sent) Enter OTP → POST action="verify_otp"
   - Step 3: (shown after verify) Table of users (username/password)


9. API ENDPOINTS (JSON responses)
===================================

login1.php (POST):
  Request: FormData (txtUserName, txtPassword, center, store_pos, o_token)
  Response: {"status":"200","message":"..."} — success
            {"status":"201","message":"Please fill user name and password to proceed."}
            {"status":"203","Token":"..."} — OTP required
            {"status":"209","Token":"...","message":"..."} — multi-device
            {"status":"4xx","message":"..."} — error

api-loginotp.php (GET):
  Request: ?Token=...
  Response: {"status":"203","message":"Login Type Not Detected."}
  GET with resend: POST {"Token":"...","status":"resend"}

api-loginotpcheck.php (POST):
  Response: {"status":"error","message":"Invalid OTP"} (unauthenticated)

forgot_password.php (POST):
  Response: {"status":"ok","msg_type":"success","message":"","mobile":"","show_otp_form":false,"wait_seconds":0,"users":[]}

logout_from_all_device.php (GET):
  Response: empty (200 OK)

newapp/auth/login (POST, JSON):
  Request: {"username":"...","password":"..."}
  Response: 405 on GET; JSON {"success":true/false,"user":{...}} on POST

14. NOTES FOR PIXEL-PERFECT RECREATION
========================================
- The form uses floating labels via CSS transform trick (not Bootstrap floating labels)
  Labels sit ABOVE the input border, created by translate3d(15px,12px,0) + background:#fff
  so the label appears to cut through the top border of the input

- The Center and POS dropdowns are d-none (Bootstrap hidden) by default
  They are presumably revealed by JS after username lookup validates user type

- The login button initially shows fa-sign-in-alt icon
  During AJAX: switches to fa-cog + fa-spin (spinning gear)
  After response: switches back to fa-sign-in-alt

- Body background is pure white (#fff) — no background image despite CSS reference
  to img/download.png (that code is commented out in source)

- SweetAlert2 dialogs replace all alert() calls for better UX
  The .swal2-popup { font-size: 0.7rem } makes them smaller than default

- The page doctype is XHTML 1.0 Transitional (legacy), but content is HTML5-compatible

- jQuery version is 2.1.4 (quite old, loaded from datagrid/jquery.min.js)

- Bootstrap 5.0.2 is loaded from css/bootstrap.min.css (local)

- FontAwesome 6.7.2 is loaded from the verbose path:
  css/font-awesome/font-awesome-6.7.2-web/css/all.min.css

- The parallel Laravel API call (newapp/auth/login) fires before the main PHP login
  but is non-blocking — the PHP session flow continues regardless of its result
